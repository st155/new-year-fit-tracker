Setup Steps

Step 1: Go to Terra Dashboard
Step 2: Navigate to Settings ‚Üí Webhooks
Step 3: Click Add Webhook Endpoint
Step 4: Enter the webhook URL from above
Step 5: Select events to subscribe to:
  • auth - User authentication events
  • activity - Workout and activity data
  • body - Body composition data
  • daily - Daily aggregate data
  • sleep - Sleep data
  • nutrition - Nutrition data
Step 6: Save the webhook configuration
Step 7: Test the webhook by:
  1. Going to Integrations page
  2. Clicking "Connect" on Terra integration
  3. Connecting any device (WHOOP, Garmin, etc.)
  4. Checking Edge Function logs for webhook events

Troubleshooting

No webhook events?
  • Verify the webhook URL is correctly configured in Terra Dashboard
  • Check that webhook is enabled and event types are selected
  • Ensure TERRA_SIGNING_SECRET is set in Supabase Edge Function secrets
  • Check Edge Function logs for any errors
  • Try disconnecting and reconnecting the device

Required Secrets

Make sure these secrets are configured in Supabase:

  • TERRA_API_KEY - Your Terra API key
  • TERRA_DEV_ID - Your Terra Developer ID
  • TERRA_SIGNING_SECRET - Your Terra webhook signing secret
Configure secrets at:
Edge Function Secrets

Testing Webhooks

Use these methods to verify your webhook is working:

Method 1: Automated Test Script

Run the test script in your browser console:

node test-terra-webhook.js

This will test the endpoint accessibility and CORS configuration.

Method 2: Manual curl Test

Test the webhook directly with curl:

curl -X POST 'https://ueykmmzmguzjppdudvef.supabase.co/functions/v1/webhook?provider=terra' \ -H 'Content-Type: application/json' \ -d '{"type":"auth","status":"success","user":{"user_id":"test-123","provider":"GARMIN"}}'
Method 3: Terra Dashboard Webhook Tester
  1. Go to Terra Dashboard
  2. Navigate to Settings ‚Üí Webhooks
  3. Find your webhook endpoint
  4. Click Test Webhook button
  5. Select event type to test
  6. Check the response and Edge Function logs
Method 4: Check Edge Function Logs

Monitor real-time webhook events:

View Webhook Logs

Look for:

  • Webhook received messages with event type
  • Signature verification results
  • Database insertion/update outcomes
  • Any error messages
Common Issues:
  • Signature verification fails: Check TERRA_SIGNING_SECRET is correctly set
  • No webhooks received: Verify webhook URL is saved in Terra Dashboard
  • Database errors: Check terra_tokens table and RLS policies
  • Connection not saved: Ensure user has completed Terra widget flow
